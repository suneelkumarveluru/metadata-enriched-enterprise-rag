# LinkedIn Post Draft

I recently started a Data + AI research prototype focused on a practical enterprise problem:

**Metadata-Enriched and Policy-Aware RAG for AI-Ready Enterprise Data Platforms**

Most RAG implementations focus on vector search and answer generation. But in enterprise environments, that is not enough.

Enterprise RAG also needs:

* Metadata awareness
* Role-based access control
* Sensitivity classification
* Source traceability
* Audit logging
* Governance compliance
* Evaluation metrics

This project explores a Governance-First RAG pattern where enterprise documents are enriched with metadata before retrieval. The prototype demonstrates how user roles, sensitivity levels, source systems, and metadata can influence what content is retrieved and what content is blocked.

The initial prototype includes:

* Synthetic enterprise document dataset
* Metadata-enriched document chunks
* Policy-aware retrieval filtering
* Unauthorized retrieval measurement
* Citation coverage measurement
* Audit event generation
* Research paper outline and experiment design

My research direction is to explore how Data + AI architecture can make RAG systems more trustworthy, auditable, and enterprise-ready.

This is part of my ongoing work around Enterprise RAG, AI-ready data platforms, lakehouse modernization, and data governance.
